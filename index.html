Краткий план доклада:

  1. Рассмотрим небольшое приложение на VUE
  2. Посмотрим как подкдючается firebase
  3. Рассмотрим работу с аутентификацией, базой данных.

Основные возможности firebase при работе с вебом:

  1. База данных в реальном времени
  2. Аутентификация
  3. Облачное хранилище данных
  4. Хостинг
  5. Пуш уведомления
  6. Куча возможностей для работы с аналитикой, работе с рекламой и многое другое, что может понадобиться при разработке мобильного приложения.

Особенность базы данных firebase - NoSQL

С высоты своего опыта отметил для себя:

Плюсы:

  1. маштабируемость
  2. быстрая работа
  3. работа в облаке
  4. хранение любых типов данных

Минусы:

  1. отсутствие четкой структуры
  2. сохранность и доступность данных
  3. поддержка - реляционные базы имеют богатый опыт и уже давно используются

Итак, если:

  1. Надо сделать простой прототип
  2. Нет сложной выборки данных
  3. Нет требованиям к типу данных
  4. Нет еще кучи дополнительных требований которые обычно появляются по ходу.

То NoSQL подходит. Так же можно использовать эти базы данных в связке.

Отдельно стоит отметить порог входа в эту технологию. Об этом собственно этот пример.
